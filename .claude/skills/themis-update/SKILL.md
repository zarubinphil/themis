# Обновление Themis

Тянет последнюю версию **логики** из GitHub и доустанавливает новое, **не трогая данные пользователя**.

## Что обновляется
- Системные пути: `.claude/` (агенты, скиллы, команды, протокол), `AGENTS.md`, `scripts/`, `cockpit/`, `bin/vision-ocr.swift`, `install.sh`, `README.md`, `.mcp.json`, `docs/`.
- Новые Python-зависимости, ffmpeg, пересборка Apple Vision OCR — через `install.sh`.

## Что НЕ трогается (данные пользователя)
- `cases/` — все дела клиентов.
- `knowledge/practice_index.md`, `knowledge/redlines.md`, `knowledge/lessons-log.md` — накопленная практика, уроки самообучения.
- Рантайм (`cockpit/.state.json`, кеши, `_baselines/`).

## Как выполнить

Запусти движок обновления:

```bash
bash scripts/update.sh
```

Скрипт сам:
1. Проверяет GitHub (`git fetch`), сравнивает версии.
2. Если есть новое — обновляет только системные пути (`git checkout origin/<branch> -- <система>`), данные не перечислены и не затрагиваются.
3. Прогоняет `install.sh` — доустанавливает новые зависимости, пересобирает OCR при изменении исходника.

После запуска кратко доложи пользователю: была ли новая версия, что обновилось, подтверди что `cases/` и `knowledge/` целы.

## Если это не git-клон
Скрипт сообщит и остановится. Тогда: склонировать свежий Themis в отдельную папку и перенести в него свои `cases/` и `knowledge/` (только их).
