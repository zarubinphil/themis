# Скилл: /askacouncil — Совет по судебной практике

Адаптация методологии Karpathy LLM Council для юридической практики.
Три охотника искали независимо — совет проверяет, фильтрует, синтезирует.

Один охотник может ошибиться. Три охотника под перекрёстным допросом — нет.

---

## Когда активируется

- После завершения всех трёх охотников (hunter_classic.md, hunter_skeptic.md, hunter_tactical.md)
- Команда: `/askacouncil {путь_к_делу}`
- Автоматически — из флоу doc-drafter, если practice.md ещё не имеет маркера

---

## Принцип работы

```
Шаг 1 — Сформулировать вопрос совета (с прочтением контекста)
Шаг 2 — Анонимное перекрёстное рецензирование (3 рецензента параллельно)
Шаг 3 — Синтез председателя → practice.md
```

Никакой дискуссии в реальном времени — только структурированные раунды.
Каждый рецензент видит АНОНИМИЗИРОВАННЫЕ отчёты, чтобы не было предвзятости
к источнику (охотник А vs Б vs В).

---

## Шаг 1. Прочти контекст и сформулируй вопрос совета

### 1.1. Прочитай файлы

```
cases/{client}/{case}/01_context/_practice/hunter_classic.md
cases/{client}/{case}/01_context/_practice/hunter_skeptic.md
cases/{client}/{case}/01_context/_practice/hunter_tactical.md
cases/{client}/{case}/01_context/knowledge-map.md  ← контекст дела
cases/{client}/{case}/_case.md
```

### 1.2. Сформулируй вопрос совета

Не пересказывай отчёты. Сформулируй нейтральный вопрос, который получат
все рецензенты:

```
Три независимых охотника исследовали судебную практику по следующим правовым
вопросам дела {клиент} / {дело}:

ПРАВОВЫЕ ВОПРОСЫ:
{список вопросов из knowledge-map.md}

КОНТЕКСТ ДЕЛА:
{стороны, суть, применимый кодекс — кратко из _case.md}

ЧТО НА КОНУ:
{какой документ готовится / какая стадия дела}

Задача совета: оценить надёжность найденной практики, выявить пробелы
и противоречия между охотниками, синтезировать финальную базу практики.
```

---

## Шаг 2. Анонимное рецензирование (3 субагента параллельно)

### 2.1. Анонимизация

Случайно назначь отчётам охотников буквы:
- Один из охотников → Отчёт А
- Другой → Отчёт Б
- Третий → Отчёт В

Запомни маппинг для шага 3 (деанонимизации). Не раскрывай его рецензентам.

### 2.2. Промпт рецензента

Запусти трёх рецензентов одновременно через Agent tool.
Каждый получает:

```
Ты — старший юрист, рецензирующий работу трёх аналитиков практики.

Дело передано тебе на оценку:

---
{вопрос совета из шага 1.2}
---

Три аналитика независимо исследовали практику и представили отчёты:

**Отчёт А:**
{содержимое}

**Отчёт Б:**
{содержимое}

**Отчёт В:**
{содержимое}

Ответь на три вопроса. Будь конкретен. Ссылайся на отчёты по буквам.

1. СИЛЬНЕЙШИЙ ОТЧЁТ: Какой отчёт наиболее надёжен и полезен для дела? Почему?
   Назови один отчёт. Объясни: что в нём сильно, что выгодно отличает.

2. СЛАБОЕ МЕСТО: У какого отчёта наибольший слепой пятно?
   Назови один отчёт. Что именно он пропустил или сделал неправильно?

3. ОБЩИЙ ПРОБЕЛ: Что пропустили ВСЕ три отчёта?
   Чего не хватает в совокупности — важный вопрос, угроза, источник?

Объём: до 250 слов. Только факты — без дипломатии.
```

### 2.3. Ожидай все три ответа

Не переходи к шагу 3, пока все три рецензента не завершили.

---

## Шаг 3. Синтез председателя

Запусти один субагент-председатель. Он получает:
- Вопрос совета (из шага 1.2)
- Все три отчёта **с раскрытыми именами** (деанонимизированные: охотник А = "Классик" и т.д.)
- Все три рецензии

```
Ты — председатель совета по судебной практике.

Три охотника независимо искали практику по делу. Три рецензента оценили
их работу. Твоя задача — синтезировать финальную базу практики.

ВОПРОС СОВЕТА:
{из шага 1.2}

ОТЧЁТЫ ОХОТНИКОВ (деанонимизированы):
**Классик (авторитетная практика):**
{hunter_classic}

**Скептик (практика против нас):**
{hunter_skeptic}

**Тактик (процессуальная практика):**
{hunter_tactical}

РЕЦЕНЗИИ:
{все три рецензии}

Составь финальный вердикт совета строго по структуре:

## Где охотники сходятся
[Что подтверждено двумя или тремя охотниками. Это высоконадёжные сигналы.]

## Где охотники расходятся
[Реальные противоречия между отчётами. Не сглаживай. Объясни, почему расходятся
и как это влияет на дело.]

## Слепые пятна, вскрытые рецензентами
[То, что выявилось только через перекрёстный анализ. Чего не заметил никто
из охотников, но рецензенты указали.]

## Уровни надёжности практики
[Каждый найденный акт — в один из трёх уровней:
Уровень 1 (Подтверждён) — упомянут 2–3 охотниками или рецензентами
Уровень 2 (Один источник) — найден одним охотником, не оспорен
Уровень 3 (Спорный) — охотники дают противоположные версии по одному вопросу]

## Итоговая рекомендация
[Что конкретно использовать в документе. Не "рассмотрите оба варианта".
Прямая рекомендация: ссылаться на X, готовить контрдовод на Y, Z — не использовать.]

## Первый шаг
[Одно действие: что сделать прямо сейчас — донайти практику, запросить у клиента,
принять как есть.]
```

## Шаг 4. Запиши итоговый practice.md

На основе вердикта председателя запиши или обнови:
`cases/{client}/{case}/01_context/practice.md`

**Инкрементально** — если файл уже содержит разделы по закрытым вопросам,
не перезаписывай их. Добавляй новые разделы и обновляй существующие.

## Шаг 5. Сохрани рабочие файлы совета

Рецензии и вердикт — для аудита и возможного оспаривания:
```
01_context/_practice/council_reviews.md   ← три анонимных рецензии
01_context/_practice/council_verdict.md   ← вердикт председателя (полный)
```

## Важные правила (из методологии Karpathy)

- **Всегда запускать всех трёх рецензентов параллельно.** Последовательный запуск
  позволяет более поздним рецензентам "дрейфовать" к ранним ответам.

- **Анонимизация — обязательна.** Если рецензенты знают, кто какой охотник,
  они будут оценивать источник, а не качество работы.

- **Председатель может не согласиться с большинством.** Если 2 из 3 рецензентов
  хвалят отчёт, но аргументация третьего сильнее — председатель следует
  за аргументом, не за голосованием.

- **Маркер обязателен.** Без строки `## СОВЕТ ЗАВЕРШЁН` в practice.md
  doc-drafter не начнёт работу.

- **Не советовать тривиальные вопросы.** Если практика по вопросу уже закрыта
  в practice.md — не запускать повторно.
