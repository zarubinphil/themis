# /init-cases — прогрев кешей по активным делам

Команда запускает `case-mapper` для всех дел, у которых нет `knowledge-map.md`.

## Порядок работы

### Шаг 1. Найди дела без knowledge-map

Прочитай `cases/_index.md`. Выбери все дела со статусом "Активно".

Для каждого дела проверь наличие файла:
```
cases/{клиент}/{дело}/01_context/knowledge-map.md
```

Раздели на две группы:
- **Нужен прогрев** — файла нет
- **Уже готово** — файл есть

Выведи список и спроси подтверждение перед запуском:

```
Найдено активных дел: N
  Нужен прогрев (N): клиент/дело, ...
  Уже готово (N): клиент/дело, ...

Запустить case-mapper для дел без карты знаний? [да/нет]
```

### Шаг 2. Запусти case-mapper параллельно

После подтверждения — запусти агента `case-mapper` для каждого дела из группы
"Нужен прогрев". Запускай параллельно, но не более 3 одновременно
(чтобы не перегружать контекст).

### Шаг 3. Отчёт

После завершения выведи:

```
=== Результат /init-cases ===

✓ Готово:
  • клиент/дело — knowledge-map.md создан, N файлов обработано

⚠ Пропущено (нет файлов в 00_intake/):
  • клиент/дело

✗ Ошибки:
  • клиент/дело — причина

Следующий шаг: запусти /init-practice для подбора практики по активным делам.
```
