# /init-practice — запуск охоты за практикой

Запускает роевых охотников для дел, у которых есть `knowledge-map.md` с маркером
`КАРТА ГОТОВА ✓`, но нет или устарел `practice.md`.

## Порядок работы

### Шаг 0. Проверь доступность сервисов поиска

До запуска охотников — проверь какие инструменты доступны:

```bash
# Проверка ScrapeGraphAI
sgai validate --json 2>/dev/null && echo "sgai: доступен" || echo "sgai: недоступен"
```

Firecrawl MCP — статус определяется при первом запросе охотника.

Объяви результат:
```
Сервисы поиска:
  Firecrawl MCP: проверяется при старте
  ScrapeGraphAI: {доступен / недоступен}
  WebSearch/WebFetch: доступны (резервный)

Приоритет поиска для охотников:
  1. Firecrawl MCP
  2. ScrapeGraphAI (если firecrawl вернул ошибку)
  3. WebSearch/WebFetch (аварийный резерв)
```

---

### Шаг 1. Найди дела без актуальной практики

Прочитай `cases/_index.md`. Выбери активные дела.

Для каждого проверь:
1. Есть ли `01_context/knowledge-map.md` с маркером `## КАРТА ГОТОВА ✓` (если нет — пропустить)
2. Есть ли `01_context/practice.md` с маркером `## СОВЕТ ЗАВЕРШЁН`

Группы:
- **Нет practice.md / нет маркера** — запустить охотников
- **Устарел (старше 30 дней, нет маркера)** — обновить
- **Актуален (маркер есть)** — пропустить

Выведи список и спроси подтверждение.

---

### Шаг 2. Определи состав охотников

Для каждого дела прочитай блок `## ОХОТНИКИ — РЕШЕНИЕ ФЕМИДЫ` в knowledge-map.md.

Если блока нет — прочитай раздел 7 (правовые вопросы) и определи уровень:
- 1 вопрос → L1 → только Тактический
- 2–3 вопроса → L2 → Классик + Тактический (или все три если оппонент — организация)
- 4+ вопросов / кассация / КАС → L3 → все три

Убедись что `01_context/practice_context.md` существует. Если нет — case-mapper не
завершил Фазу 4.5. Предупреди пользователя и предложи повторить case-mapper.

---

### Шаг 3. Запусти охотников параллельно

Запускай всех нужных охотников одновременно через Agent tool:
- `practice-hunter-classic` — авторитетная иерархия (L2/L3)
- `practice-hunter-skeptic` — практика против нас (L2/L3)
- `practice-hunter-tactical` — процессуальная практика (все уровни)

Каждому передавай путь к делу: `cases/{client}/{case}/`

---

### Шаг 4. После завершения охотников

Для L2/L3: запусти `/askacouncil {путь}` → получишь `practice.md` с маркером.
Для L1: тактик роя пишет `practice.md` напрямую с маркером.

Запусти `archivist` с путём к делу → пополнение базы знаний.

---

### Шаг 5. Отчёт

```
=== Результат /init-practice ===

✓ Готово: клиент/дело — practice.md создан, маркер проставлен
⚠ Частично: клиент/дело — охотники завершены, нужен /askacouncil
↷ Пропущено (актуально): клиент/дело

Следующий шаг: L2/L3 → /council → positions.md → doc-drafter
```
