# Команда: /finalize

Финализирует судебный документ: конвертирует в PDF и накладывает подпись.

## Когда использовать

После того как:
1. Кони (doc-reviewer) вынес вердикт ГОТОВ К ПОДАЧЕ
2. Ты проверил .docx вручную (опционально отредактировал и сохранил)
3. Готов получить финальный PDF с подписью

## Использование

```
/finalize {путь_к_docx}
/finalize {клиент}/{дело}    ← найдёт последний docx в 02_hearings/ автоматически
```

## Что делает

1. Находит `.docx` — в `02_hearings/{последнее заседание}/` если путь не указан
2. Проверяет наличие подписи в `cases/_assets/подпись.png`
3. Конвертирует `.docx` → `.pdf` через Microsoft Word
4. Накладывает PNG подписи на последнюю страницу в блоке подписи
5. Сохраняет `.pdf` рядом с `.docx` в той же папке заседания
6. Обновляет `_event.md` — отметить документ как подписан

## Структура папки заседания после финализации

```
02_hearings/ДД-ММ-ГГГГ_название/
  _event.md
  {документ}.md       ← рабочий Markdown (источник)
  {документ}.docx     ← финальный Word (для правок)
  {документ}.pdf      ← финальный PDF с подписью (для подачи)
```

## Если подписи нет

```
⚠ Подпись не найдена: cases/_assets/подпись.png
Добавь PNG подписи → повтори /finalize
```

## Только PDF без подписи

```
/finalize {путь} --no-sign
```
