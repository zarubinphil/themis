# Команда: /draft

Составляет процессуальный документ через скилл `doc-drafter`.

## Что делает

Активирует скилл `doc-drafter` (Сперанский). Скилл:

1. Находит дело в `cases/_index.md`
2. Проверяет маркеры консенсуса (КАРТА ГОТОВА / СОВЕТ ЗАВЕРШЁН / СОГЛАСОВАНО СОВЕТОМ) — если нет, СТОП
3. Составляет документ на основе `knowledge-map.md`, `practice.md`, `positions.md`
4. Сохраняет черновик в `03_drafts/{документ}_v1.md`
5. Запускает `doc-reviewer` (Кони) → 7 блоков проверки

**Цикл итерации (до 2 раундов):**
- Кони: **ГОТОВ** → генерирует `.docx` → сохраняет в `02_hearings/ДД-ММ-ГГГГ_название/` → конец
- Кони: **ТРЕБУЕТ ПРАВОК** → Сперанский вносит правки → сохраняет `_v2.md` → Кони проверяет снова
- Кони: **КРИТИЧЕСКИЕ ОШИБКИ** → СТОП, сообщить пользователю что именно исправить вручную
- После 2 раундов без одобрения → СТОП, вынести на решение пользователя

Каждое правовое утверждение содержит источник: `[ст. 333 ГК РФ]`, `[Пленум ВС РФ от ДД.ММ.ГГГГ № X, п. Y]`.

## Использование

```
/draft {тип документа} для {клиент}/{дело}
```

**Примеры:**
```
/draft возражения для rakhimkulova/alimenty-izmenenie-2026
/draft иск для ivanov-ivan/razdel-imushchestva-2026
/draft апелляционная жалоба для belavina-rv/apellyaciya-2-434-2025
```

Если маркеры консенсуса не проставлены — скилл остановится и укажет что запустить.
