# Команда: /new-event

Создаёт папку заседания или события по делу с шаблоном `_event.md`.

## Использование

```
/new-event {клиент} {дело} {дата} {название}
```

**Примеры:**
```
/new-event romashka-ooo vs-supplier-2024 2025-06-10 osnovnoe-zasedanie
/new-event ivanov-sergey razvod-2025 2025-07-15 predvaritelnoe
/new-event techstroy-ao bankrotstvo-2025 2025-08-01 apellyaciya
```

---

## Что делает

### 1. Создаёт папку события

```
cases/{клиент}/{дело}/02_hearings/{дата}_{название}/
└── _event.md
```

### 2. Создаёт `_event.md` по шаблону

```markdown
# {Тип события} {дата} | {дело}

**Суд:** {из _case.md}
**Номер дела:** {из _case.md}
**Судья:** {из _case.md}

---

## ДО СОБЫТИЯ

### Задача на заседание
_Что нужно доказать / добиться_

### Позиция оппонента (ожидаемая)
_Что скорее всего скажет оппонент_

### Наши контраргументы
_Готовые ответы на доводы оппонента_

### Документы для подготовки
- [ ] {документ 1}
- [ ] {документ 2}

### Что берём в суд
- [ ] {документ — оригинал/копия}

---

## ПОСЛЕ СОБЫТИЯ

### Что произошло
_Заполняется после заседания_

### Позиция суда
_Вопросы судьи, промежуточные выводы, процессуальные решения_

### Что поручил суд
_Предоставить документы, дать пояснения, и т.д._

### Следующее заседание
- **Дата:** —
- **Тип:** —

### Что готовить
- [ ] {следующий документ}
```

### 3. Обновляет `_case.md`

- Поле "Следующее заседание" → дата события
- Журнал событий → добавляет строку с датой и типом события

### 4. Выводит итог

```
✓ Событие создано: 02_hearings/{дата}_{название}/
  Заполни _event.md: задача, ожидаемая позиция оппонента
  Запусти doc-drafter для подготовки документов к заседанию
```
